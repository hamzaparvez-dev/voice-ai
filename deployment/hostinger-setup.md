# Hostinger KVM2 Deployment Guide

Deploy GenuineStack Japan (Dograh) on Hostinger KVM2 for TBIS demo and ICMG presentation.

## Prerequisites

| Requirement | Hostinger KVM2 |
|-------------|------------------|
| vCPU | 2+ cores ✅ |
| RAM | 8 GB minimum ✅ |
| Storage | 20 GB+ ✅ |
| OS | Ubuntu 22.04+ ✅ |
| Docker | Docker + Docker Compose ✅ |

## Step 1: SSH into Server

```bash
ssh root@your-server-ip
```

## Step 2: Install Docker (if not present)

```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER
# Log out and back in for group changes
```

## Step 3: Clone and Deploy

```bash
# Clone GenuineStack Japan repo (or upload files)
git clone <your-repo-url> genuinestack-japan
cd genuinestack-japan/dograh

# One-command deploy
chmod +x scripts/start_docker.sh
./scripts/start_docker.sh
```

**Alternative — direct curl deploy (no clone):**

```bash
curl -o docker-compose.yaml https://raw.githubusercontent.com/dograh-hq/dograh/main/docker-compose.yaml && \
curl -o start_docker.sh https://raw.githubusercontent.com/dograh-hq/dograh/main/scripts/start_docker.sh && \
chmod +x start_docker.sh && \
./start_docker.sh
```

First startup takes 2-3 minutes to pull Docker images. Open `http://your-server-ip:3010`.

## Step 4: Configure Environment

Edit `.env` in the dograh directory:

```env
# Core
PUBLIC_HOST=your-server-ip-or-domain
BACKEND_API_ENDPOINT=http://your-server-ip:8000
TURN_HOST=your-server-ip

# Telemetry (recommended for enterprise)
ENABLE_TELEMETRY=false

# Secrets (auto-generated by start_docker.sh, or set manually)
OSS_JWT_SECRET=<openssl rand -hex 32>
POSTGRES_PASSWORD=<strong-password>
REDIS_PASSWORD=<strong-password>
MINIO_ROOT_USER=dograh
MINIO_ROOT_PASSWORD=<strong-password>

# Optional: Bring Your Own Keys
# OPENAI_API_KEY=sk-...
# DEEPGRAM_API_KEY=...
```

## Step 5: Open Firewall Ports

```bash
# UFW example
sudo ufw allow 3010/tcp   # UI Dashboard
sudo ufw allow 8000/tcp   # API
sudo ufw allow 80/tcp     # HTTP
sudo ufw allow 443/tcp    # HTTPS
sudo ufw allow 3478/tcp   # TURN
sudo ufw allow 3478/udp   # TURN
sudo ufw enable
```

## Step 6: Custom Domain + SSL (Production)

```bash
cd dograh
./scripts/setup_custom_domain.sh yourdomain.com
```

Requires DNS A record pointing to your server IP.

## Step 7: Import TBIS Agent

1. Open `http://your-server-ip:3010`
2. Create account / sign in
3. Go to **Workflows** → **Upload Agent Definition**
4. Upload `workflows/tbis-bilingual-agent.json` from this repo
5. Go to **Files** → upload all files from `knowledge-base/`
6. In workflow editor, attach document UUIDs to agent nodes
7. **Validate** → **Publish**

## Step 8: Configure Models

Go to **Model Configurations** (`/model-configurations`):

| Component | Recommended Setting |
|-----------|---------------------|
| STT (Transcriber) | `multi` or Japanese + English |
| TTS (Voice) | Japanese + English voices |
| LLM | OpenAI GPT-4o or Dograh stack |

## Step 9: Test Web Call

1. Open TBIS workflow in editor
2. Click **Web Call**
3. Test in English and Japanese
4. Verify Keigo responses and knowledge base answers

## WebRTC / TURN Configuration

For voice calls to work remotely, configure TURN:

```env
TURN_HOST=your-public-ip-or-domain
TURN_SECRET=<openssl rand -hex 32>
```

See `dograh/deploy/hostinger/.env.example` for Hostinger-specific Traefik setup.

## Monitoring

```bash
# Live API logs
docker compose logs -f api

# All services
docker compose logs -f

# Restart
docker compose down && docker compose up -d

# Backup database
docker compose exec postgres pg_dump -U postgres postgres > backup.sql
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| UI not loading | Check port 3010 open, `docker compose ps` |
| Web call no audio | Verify TURN_HOST and port 3478 UDP/TCP |
| KB not answering | Confirm documents processed, attach UUIDs to nodes |
| Japanese not working | Set STT language to multi, verify TTS Japanese voice |
